# 方法（Method）

## 1. 问题定义与动机

多模态大语言模型（Multimodal Large Language Model, MLLM）在视觉问答任务中普遍存在**对象幻觉**（Object Hallucination）问题：模型会声称图像中存在实际上并不存在的对象。这一现象在遥感视觉语言模型中尤为突出，因为遥感图像背景复杂、目标密集，模型更容易受到语言先验的干扰而忽视视觉证据。

本工作以 GeoChat（基于 Vicuna-7B 的遥感视觉语言模型）为研究对象，在扩展的 POPE 基准上系统分析幻觉现象的成因，并提出一种**无需重新训练的推理期注意力分段重加权方法**来缓解幻觉。

### 1.1 评测基准的选择：为何使用 POPE

目前幻觉评测领域存在多种基准。早期工作常用 **CHAIR**（Caption Hallucination Assessment with Image Relevance），其核心思路是让模型对图像生成开放式描述，再统计描述中出现的幻觉物体比例。CHAIR 的局限在于：

1. **评测维度单一**：仅关注"物体是否存在"这一维度，无法评测颜色、数量、位置等更细粒度的幻觉类型
2. **依赖开放生成**：开放式描述的长度和内容难以控制，评测结果受采样策略影响较大
3. **迁移性差**：CHAIR 依赖 COCO 物体标注，难以直接迁移至遥感等垂直领域

相比之下，**POPE**（Polling-based Object Probing Evaluation）将幻觉评测转化为结构化的 Yes/No 二元问答，具备以下优势：

- **评测可控**：每道题只需输出 Yes/No，消除了生成随机性对评测结果的干扰
- **采样策略丰富**：支持随机（Random）、流行度（Popular）、对抗（Adversarial）三种负样本采样策略，能够更全面地揭示模型的不同幻觉模式
- **易于扩展**：问题模板统一，稍加修改即可扩展至新领域和新幻觉维度

更重要的是，**在遥感视觉语言理解领域，目前尚缺乏系统性的幻觉评测框架**。现有遥感 VLM 的评测主要集中于正确率和 BLEU 等指标，对幻觉问题缺乏针对性分析。本工作基于 POPE 的结构化范式，将其扩展至遥感场景，填补这一空白。

### 1.2 多维幻觉评测体系

幻觉并不仅限于"物体是否存在"这一维度。模型可能在物体识别正确的同时，在颜色、数量、空间位置等属性上产生幻觉。为此，本工作设计了四种互补的 POPE 变体，从不同维度对模型的幻觉能力进行系统评测：

#### 变体一：对象存在性幻觉（Existence POPE）

最经典的 POPE 形式。给定图像，询问某一物体是否存在，例如"Is there a runway in the image?"。负样本采用三种采样策略：

- **随机采样**（Random）：从词汇表中随机选取与图像无关的物体
- **流行度采样**（Popular）：选取数据集中最常见但当前图像不含的物体，测试模型是否存在对高频对象的先验偏见
- **对抗采样**（Adversarial）：选取与图像中已有对象视觉相似、场景语义相关或常共现的对象，设计最具迷惑性的负样本

#### 变体二：颜色属性幻觉（Color POPE）

在图像中的对象存在的前提下，进一步询问其颜色属性，例如"Is the roof red?"。负样本覆盖四种检测角度：

- **虚构性幻觉**：对图像中根本不存在的对象询问颜色
- **主色冲突幻觉**：对已有对象提供色彩理论上相反的颜色描述（如将红色物体问成蓝色）
- **语言先验诱导幻觉**：构造与图像实际颜色冲突、但符合物体常识颜色先验的问题（如将绿色香蕉问成黄色）
- **颜色混淆幻觉**：在多对象场景中，将对象 A 的颜色张冠李戴地问成对象 B 的颜色

若图像描述中不含颜色信息，则该维度标记为"no color info"并跳过。

#### 变体三：数量属性幻觉（Quantity POPE）

询问图像中特定对象的数量，例如"Are there three airplanes in the image?"。负样本覆盖三种检测角度：

- **虚构性数量幻觉**：对图像中根本不存在的对象询问数量
- **基础计数错误幻觉**：对已有对象给出严重偏离真实数量的错误数字
- **多类别数量干扰幻觉**：在多类别场景中，将对象 A 的数量混淆为对象 B 的数量

若图像描述中不含明确数量信息（仅有"many"、"some"等模糊表述），则该维度标记为"no quantity info"并跳过。

#### 变体四：空间位置关系幻觉（Spatial POPE）

询问图像中对象间的空间位置关系，例如"Is the runway to the left of the terminal building?"。负样本覆盖三种检测角度：

- **虚构性位置幻觉**：对图像中不存在的对象询问位置关系
- **相对位置反转幻觉**：将已有对象的上下/左右/前后关系刻意颠倒后提问
- **空间区域错误幻觉**：将对象实际所在的空间区域（如左上角）替换为错误区域（如右下角）后提问

若图像描述中不含明确空间位置信息，则该维度标记为"no position info"并跳过。

---

通过上述四种变体的组合，本工作构建了一个覆盖**对象存在性、颜色属性、数量属性、空间位置关系**四个维度的遥感视觉语言模型幻觉评测框架，为后续注意力重加权方法的分析提供多粒度的评测支撑。

POPE 任务的统一形式为二元视觉问答：给定图像 $I$ 和问题 $Q$，模型需输出"Yes"或"No"。

## 2. 幻觉注意力头识别

### 2.1 阶段一：基线推理与样本划分

首先使用 GeoChat 在 POPE 数据集子集上进行无任何干预的基线推理，对每条样本记录预测结果 $\hat{y}$ 与真实标签 $y$。按照预测正确与否，将全部样本划分为两组：

- **正确样本集** $\mathcal{C} = \{s_i \mid \hat{y}_i = y_i\}$：模型回答正确，未发生幻觉
- **错误样本集** $\mathcal{E} = \{s_i \mid \hat{y}_i \neq y_i\}$：模型回答错误，发生幻觉

两组样本将作为后续归因分析的对照组与实验组。

### 2.2 阶段二：逐头消融归因

对 $\mathcal{C}$ 和 $\mathcal{E}$ 中的每条样本，逐一计算模型中每个注意力头对最终预测 token 概率的影响力。

#### 零消融机制

对于第 $l$ 层第 $h$ 个注意力头，采用**前向钩子（Forward Pre-Hook）**的方式实施零消融：在注意力输出经过输出投影层 $W_O$（即 `o_proj`）之前，将对应该 Head 的维度切片强制置零：

$$\text{hidden}_{:,:,\, h \cdot d_h:(h+1) \cdot d_h} \leftarrow \mathbf{0}$$

其中 $d_h = d_{\text{model}} / H$ 为每个 Head 的维度大小（GeoChat 中 $d_{\text{model}}=4096$，$H=32$，故 $d_h=128$）。这一操作等价于完全切断该 Head 对后续计算的贡献，同时不影响其他 Head。

#### 影响力分数定义

对于样本 $s_i$，定义第 $(l, h)$ 号注意力头的**影响力分数**为：

$$\text{Inf}_i(l, h) = P_{\text{orig}}(y^* \mid s_i) - P_{\text{ablated}}(y^* \mid s_i; \text{zero}(l,h))$$

其中 $y^*$ 为模型在未消融状态下预测的最高概率 token，$P_{\text{orig}}$ 和 $P_{\text{ablated}}$ 分别为消融前后对该 token 的 softmax 输出概率。

此定义的直觉如下：
- $\text{Inf}_i(l,h) > 0$：消融该 Head 后预测概率下降，说明该 Head **正向促进**了当前预测
- $\text{Inf}_i(l,h) < 0$：消融后概率反而上升，说明该 Head 对当前预测具有**抑制**作用
- $\text{Inf}_i(l,h) \approx 0$：该 Head 对当前预测几乎无贡献

对 $\mathcal{C}$ 和 $\mathcal{E}$ 中的全部样本重复上述过程，每个样本产生一个 $L \times H$（32×32）的影响力矩阵。

### 2.3 阶段三：差异化聚合与幻觉头排序

分别对正确样本集和错误样本集的影响力矩阵在样本维度上取均值：

$$\bar{M}_{\mathcal{C}}(l,h) = \frac{1}{|\mathcal{C}|}\sum_{i \in \mathcal{C}} \text{Inf}_i(l,h), \quad \bar{M}_{\mathcal{E}}(l,h) = \frac{1}{|\mathcal{E}|}\sum_{i \in \mathcal{E}} \text{Inf}_i(l,h)$$

定义每个注意力头的**幻觉得分**（Hallucination Score）：

$$S(l,h) = \bar{M}_{\mathcal{E}}(l,h) - \bar{M}_{\mathcal{C}}(l,h)$$

**语义解释**：$S(l,h)$ 越高，说明该 Head 在幻觉样本中对错误预测的正向促进程度，显著高于其在正确样本中的贡献——即该 Head 是驱动模型产生幻觉的关键组件。反之，得分较低的 Head 在正确推理中发挥更大作用，不应被干预。

按 $S(l,h)$ 降序对全部 $L \times H = 1024$ 个注意力头排序，取前 $N$ 个作为**幻觉头**（Hallucination Heads），用于后续推理期的注意力重加权干预。$N$ 为超参数，在实验中于 $\{20, 40, 60, 80, 100\}$ 上搜索。

## 3. 输入序列分段建模

### 3.1 Token 序列的语义分段

GeoChat 在推理时，完整的 KV 序列由四个具有不同语义角色的段构成：

$$\mathbf{X} = \underbrace{[\,x_1,\ldots,x_{p_s}\,]}_{\text{SYS}} \;\|\; \underbrace{[\,x_{p_s+1},\ldots,x_{p_s+P}\,]}_{\text{VIS}} \;\|\; \underbrace{[\,x_{p_s+P+1},\ldots,x_{p_e}\,]}_{\text{INST}} \;\|\; \underbrace{[\,x_{p_e+1},\ldots\,]}_{\text{RESP}}$$

| 段 | 内容 | 作用 |
|:---|:---|:---|
| **SYS** | System Prompt | 定义模型角色、对话格式与输出规范 |
| **VIS** | 图像 Patch Token | 承载视觉内容的全部感知信息 |
| **INST** | 用户问题（Instruction）| 指定具体查询目标 |
| **RESP** | 已生成的回复 Token | 推理过程中的自回归历史（首步为空）|

### 3.2 图像 Token 数量的精确计算

GeoChat 的视觉编码器将输入图像统一 resize 至 $504 \times 504$ 像素，经 ViT patch encoder（patch size = 14）处理后，图像被编码为：

$$P = \left\lfloor \frac{504}{14} \right\rfloor \times \left\lfloor \frac{504}{14} \right\rfloor = 36 \times 36 = 1296 \text{ tokens}$$

> **关键修正**：早期实现中图像 token 数量被错误地硬编码为 576（对应 $224\times224$ 分辨率的 ViT 配置），导致 VIS 段边界计算错误，INST 段被大量错误掩码，推理干预完全失效。本工作通过动态获取视觉编码器配置来精确计算该值：
>
> ```python
> actual_img_tokens = (H // patch_size) * (W // patch_size)  # = 1296
> ```

### 3.3 各段边界的动态定位

各段在展开后 token 序列中的精确边界计算如下：

- $p_s$（VIS 段起始）：在原始 `input_ids` 中定位 `IMAGE_TOKEN_INDEX` 占位符的位置（自动检测）
- $p_s + P$（VIS 段结束）：$p_s + 1296$
- $p_e$（INST 段结束 / RESP 段起始）：$\text{input\_ids.shape}[1] - 1 + 1296$，即将单个图像占位符展开为 1296 个 patch token 后，原始 prompt 的末尾位置

## 4. 推理期注意力分段重加权

### 4.1 重加权机制

对于识别出的 $N$ 个幻觉头中的每一个 $(l, h)$，在 Softmax 归一化**之前**对注意力 logits 按语义段施加可控的缩放因子 $\gamma$：

$$\tilde{A}_{l,h}[q, k] = \gamma_{\text{seg}(k)} \cdot A_{l,h}[q, k]$$

其中 $\text{seg}(k)$ 表示 key position $k$ 所属的语义段，对应的缩放因子为：

$$\gamma_{\text{seg}} = \begin{cases} \gamma_{\text{sys}} & k \in \text{SYS} \\ \gamma_{\text{vis}} & k \in \text{VIS} \\ \gamma_{\text{inst}} & k \in \text{INST} \\ \gamma_{\text{resp}} & k \in \text{RESP} \end{cases}$$

重加权后对该 Head 的注意力权重重新归一化，以保证数值稳定性：

$$\hat{A}_{l,h}[q,:] = \frac{\tilde{A}_{l,h}[q,:]}{\displaystyle\sum_{k'} \tilde{A}_{l,h}[q,k'] + \epsilon}$$

**重要设计原则**：重加权**仅作用于** $N$ 个幻觉头，模型中其余 $L \times H - N$ 个正常注意力头保持完全不变。这确保了模型的语言能力、对话格式理解和输出规范不受任何影响。

### 4.2 自适应激活阈值

为避免对所有 token 位置无差别地强制干预，引入**自适应阈值机制**：在每个生成步，仅当幻觉头的文本注意力占比超过阈值 $\tau$ 时才触发重加权。

对于幻觉头 $(l, h)$，在最后一个 query position $q = q_\text{len} - 1$ 处计算各段的注意力总量：

$$M_{\text{seg}} = \sum_{k \in \text{seg}} A_{l,h}[q_\text{last}, k]$$

文本注意力占比定义为：

$$r = \frac{M_{\text{sys}} + M_{\text{inst}}}{M_{\text{sys}} + M_{\text{vis}} + M_{\text{inst}} + M_{\text{resp}}}$$

仅当 $r \geq \tau$ 时执行重加权，否则跳过该 Head 本步的干预：

$$\text{apply\_reweight}(l,h) = \mathbb{1}[\,r \geq \tau\,]$$

此设计的动机是：若某幻觉头在当前步已经主要关注图像（$r$ 较小），则其行为已经合理，无需强制干预；只有当该 Head 过度关注文本时，才需要将其注意力重新引导至视觉内容。

阈值 $\tau$ 在 $\{0.1, 0.2, 0.4, 0.6\}$ 上搜索。

## 5. 关键发现：指令诱导幻觉

### 5.1 现象

通过对 $\gamma_{\text{inst}}$ 的系统性消融实验，我们发现了一个反直觉的规律：

| $\gamma_{\text{inst}}$ | POPE 准确率 |
|:---:|:---:|
| 1.0（无干预基线） | 58.63% |
| 0.1 | 72.18% |
| 0.0 | 74.60% |

在幻觉头中，将 INST 段的注意力权重压制至 0（$\gamma_{\text{inst}} = 0$）可以带来约 **16 个百分点**的准确率提升。

### 5.2 机制分析

这一现象揭示了 POPE 任务中的**指令诱导幻觉**（Instruction-Induced Hallucination）机制：

POPE 问题的模板为"Is there a [object] in the image?"，问题文本中**显式地包含了被查询物体的名称**。幻觉头对该名词具有强烈的文本注意力偏好，使得模型在生成回答时被问题中的物体名词"锚定"，倾向于输出"Yes"，即便图像中并不存在该物体。

将 $\gamma_{\text{inst}} = 0$ 作用于幻觉头后：

- **幻觉头**：不再被问题中的物体名词干扰，注意力完全重新分配至 SYS 和 VIS 段，基于视觉内容作出判断
- **其余正常 Head**（共 $1024 - N$ 个）：完整读取 INST 段，保证模型理解"需要回答 Yes/No"的格式要求

两路信息并行传递，互不干扰，在抑制幻觉的同时不破坏输出的格式合法性。

## 6. 超参数搜索与最终配置

在固定 $\gamma_{\text{sys}} = 1.0$，$\gamma_{\text{inst}} = 0.0$，$\gamma_{\text{resp}} = 1.0$ 的基础上，对幻觉头数量 $N$、激活阈值 $\tau$、视觉段缩放因子 $\gamma_{\text{vis}}$ 进行网格搜索。

各参数的搜索区间为：
- $N \in \{20, 40, 60, 80, 100\}$
- $\tau \in \{0.1, 0.2, 0.4, 0.6\}$
- $\gamma_{\text{vis}} \in \{0.0, 0.2, 0.5, 0.7, 1.0\}$

最优配置如下：

| 超参数 | 最优值 |
|:---|:---:|
| $\gamma_{\text{sys}}$ | 1.0 |
| $\gamma_{\text{vis}}$ | 1.0 |
| $\gamma_{\text{inst}}$ | 0.0 |
| $\gamma_{\text{resp}}$ | 1.0 |
| 幻觉头数量 $N$ | 40 |
| 激活阈值 $\tau$ | 0.4 |

该配置在 POPE 基准上达到 **77.96%** 的准确率，相比无干预基线（58.63%）提升 **+19.33 个百分点**，且全程无需对模型进行任何微调。
